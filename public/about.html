How scryb works

scryb turns audio and video recordings into clean, readable text using a combination of local AI models and Groq's cloud transcription API — all running from your machine.

Overview

Groq Whisper (primary)

Whisper Large v3 Turbo runs on Groq's cloud. Transcribes a 1-hour meeting in ~10 seconds with near-perfect accuracy.

whisper.cpp (fallback)

Local model runs on your CPU — no internet needed. Used when Groq is unavailable or the file exceeds 24 MB.

Llama 3.1 refinement

After local transcription, Llama 3.1 fixes grammar, removes artifacts and improves readability. Skipped for Groq Whisper.

100% private

Files are processed locally and deleted right after transcription. Your audio never leaves your machine except to Groq's API.

Transcription pipeline

What happens when you click Transcribe

1

Upload & validate

Files are received by the Express server via multipart form upload. Unsupported formats are rejected immediately.

2

Convert to WAV

FFmpeg converts every file to 16 kHz mono PCM WAV — the format whisper models expect. Saved to %TEMP%\whisper-transcricao\.

3

Groq Whisper Large primary

The WAV is re-encoded to OGG (smaller) and sent to Groq's API with your language and context prompt. Returns the transcript in seconds.

4

whisper.cpp local fallback

If Groq is unavailable or the file is >24 MB, whisper-cli.exe runs locally with anti-hallucination flags (-et 2.8 -lpt -0.5).

5

Clean & refine

Noise markers, subtitle hallucinations and filler word loops are removed. For local transcriptions, Llama 3.1 does a final grammar pass.

6

Result delivered

The transcript appears in the UI. Temp files are deleted. The completed job is saved to storage/jobs.json for recovery on restart.

Reliability

Job queue & persistence

Each transcription request is a job tracked in memory and on disk. The UI polls for updates every 1.2 seconds.

If you close the browser tab or reload while transcription is running, the job keeps processing in the background. When you reopen the page, it automatically reconnects and shows the result.

On server restart, all completed jobs are restored from storage/jobs.json (up to 100 jobs).

Craig bot integration

Transcribing Discord calls

Craig is a Discord bot that records multi-track audio from voice channels — one .flac file per participant.

Use the Craig folder drop zone to select the entire Craig export folder. scryb transcribes each track separately and labels the output by filename, so you can tell who said what.

Tip: Fill in the Context field with the names of participants and any technical terms used in the call. This is passed to Whisper as a prompt hint and significantly improves recognition of names and jargon.

Setup

Groq API key

Groq offers a generous free tier: 7,200 minutes per day (120 hours). For personal use, this is effectively unlimited.

Get a free key at console.groq.com/keys, then paste it in the ⚙️ Settings panel. The key is stored locally in your user profile — never shared with anyone.

Without a Groq key: scryb falls back to the local whisper.cpp model. Transcription still works, but is slower and less accurate.
